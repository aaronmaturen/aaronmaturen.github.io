Galactic Archives Series

A comprehensive guide to building efficient data sources in Angular using the Star Wars API.

This series explores advanced patterns for handling paginated data in Angular applications. Using the Star Wars API (SWAPI) as our data source, we'll build a robust, reusable solution for displaying and navigating large datasets.

What You'll Learn

  • Creating custom data sources in Angular
  • Handling paginated APIs efficiently
  • Implementing virtual scrolling
  • Managing complex state with RxJS
  • Building reusable, testable components

Table of Contents

Foundation Phase

  1. Introduction - Introduces the DataSource pattern concept and outlines the plan to build a Star Wars data explorer application called "Galactic Archives"
  2. Basic Project Setup - Creates the Angular v18 project with standalone components, sets up Angular Material and Tailwind CSS with tw- prefix, and implements a basic app shell
  3. Code Quality Setup - Implements Husky and lint-staged for pre-commit hooks to ensure code quality
  4. Testing Setup - Configures Jest for unit/integration testing and Playwright for E2E testing

Architecture Phase

  1. Feature-Based Architecture - Reorganizes the application using a feature-based architecture with core, features, shared, and models directories
  2. Star Wars API Service - Implements the StarWarsService to interact with the SWAPI API
  3. API Mocking with MSW - Sets up Mock Service Worker for API mocking during development and testing

Core DataSource Implementation Phase

  1. DataSource Foundation - Implements the core GalacticDataSource class extending Angular's DataSource
  2. Implementing Pagination - Adds pagination to the DataSource and implements MatTable with MatPaginator
  3. Adding Sorting - Enhances the DataSource with sorting capabilities using MatSort
  4. Implementing Filtering - Adds filtering functionality to the DataSource with debouncing

UI Enhancement Phase

  1. Star Wars Theming - Customizes the Angular Material theme with Star Wars-inspired colors
  2. Responsive Design - Makes the application fully responsive across different device sizes
  3. Keyboard Navigation - Improves accessibility with keyboard navigation support
  4. High Contrast Mode - Adds support for high contrast mode for visually impaired users
  5. Loading States and UX - Enhances user experience with better loading states and indicators

Testing and Deployment Phase

  1. Testing DataSource - Implements comprehensive tests for the DataSource using Jest and MSW
  2. Deployment Optimization - Optimizes the application for production with bundle analysis, service worker implementation, and advanced caching
  3. CI/CD with GitHub Actions - Sets up continuous integration and deployment pipelines
  4. Future Directions and Advanced Patterns - Explores hypothetical improvements including Angular Signals, NgRx integration, advanced caching strategies, and performance optimizations

Prerequisites

  • Basic knowledge of Angular and TypeScript
  • Familiarity with RxJS observables
  • Understanding of HTTP client operations
